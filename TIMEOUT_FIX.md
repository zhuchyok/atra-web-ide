# Исправление: Таймаут в ExtendedThinking

## Проблема

Тест завершился с пустым результатом из-за таймаутов:
- ExtendedThinking использует таймаут 120 секунд
- Генерации deepseek-r1-distill-llama:70b занимают 121-123 секунды
- Запрос отменяется по таймауту ДО завершения генерации
- Результат: пустой

## Временная шкала ошибки

```
00:37:45 - Запрос отправлен к MLX Server
00:39:45 - ReadTimeout (120 секунд прошло) ❌
00:39:47 - Генерация завершена (121.59 секунд) ✅
```

MLX Server успешно генерирует, но ExtendedThinking уже отменил запрос.

## Решение

Увеличен таймаут в `extended_thinking.py`:
- Было: `timeout=120.0` (2 минуты)
- Стало: `timeout=300.0` (5 минут)

Это дает запас для генераций, которые могут занимать до 125 секунд.

## Файлы изменены

- `knowledge_os/app/extended_thinking.py`:
  - Основной запрос: `timeout=300.0`
  - Fallback запросы: `timeout=300.0`
