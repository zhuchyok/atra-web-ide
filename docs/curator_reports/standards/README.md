# Эталоны ответов куратора

Здесь хранятся **эталонные ответы** на типовые запросы — «как бы ответил куратор (Cursor-агент)». Используются при разборе отчётов Victoria: сравнение output с эталоном, выявление расхождений, передача лучших формулировок в knowledge_nodes.

**Формат файла:** один markdown-файл на тип запроса (или группу). Внутри:
- **Цель (goal)** — точная формулировка запроса.
- **Эталонный ответ** — текст или ключевые пункты.
- **Критерии** — что проверять (полнота, тон, структура).
- Опционально: дата, источник (FINDINGS, прогон).

**Примеры имён:** `what_can_you_do.md`, `greeting.md`, `status_project.md`, `list_files.md`, `one_line_code.md`.

---

## Добавление эталона в базу знаний (RAG)

Чтобы Victoria подтягивала эталонный ответ через RAG: добавить узел в **knowledge_nodes** (Knowledge OS). Варианты:

1. **Скрипт (рекомендуется):** `DATABASE_URL=... python3 scripts/curator_add_standard_to_knowledge.py` — добавляет в домен `curator_standards` все 5 эталонов: что ты умеешь, привет, статус проекта, список файлов, одна строка кода (Python дата). Идемпотентно (при наличии узла — пропуск).
2. **API:** `POST http://localhost:8002/knowledge` (knowledge_rest) с телом `{"domain": "atra-web-ide", "content": "Вопрос: ... Ответ: ...", "confidence_score": 0.95, "metadata": {"source": "curator_standard"}}`. Заголовок авторизации по конфигу API (JWT/API_KEY).
3. **Миграция/seed:** по аналогии с `knowledge_os/db/seed_knowledge.sql`.

После добавления RAG будет возвращать этот контент при релевантных запросах.
