# Чеклист куратора при разборе отчётов Victoria

**Кто:** агент в Cursor (куратор и наставник). **Когда:** после прогона `curator_send_tasks_to_victoria.py` или по запросу «проверь Victoria».

---

## 1. Цепочка и маршрутизация

- [ ] Запрос попал туда, куда нужно: простой («покажи файлы») → Veronica или quick_data; сложный/исполнение → Victoria Enhanced.
- [ ] В `knowledge.execution_trace` видно: `routed_to`, `task_type`, при делегировании — `delegated_to`.
- [ ] Нет лишнего вызова LLM для информационных фраз («что ты умеешь», «кто ты») — быстрый путь без модели.

## 2. Качество ответа (как бы сделал я)

- [ ] **Полнота:** ответ закрывает вопрос, без «общих фраз» вместо конкретики.
- [ ] **Точность:** нет выдуманных фактов, путей, имён; при сомнении — явное «не знаю» или уточнение.
- [ ] **Формат:** структурированно (списки, заголовки), без лишней воды; для кода — рабочий фрагмент.
- [ ] **Контекст:** учтён project_context и история, если передана; при решении Совета — ответ опирается на директиву.

## 3. Эталон «как я»

Для выбранных задач куратор фиксирует эталонный ответ (как бы ответил сам):

- Цель (goal).
- Эталонный текст (или ключевые пункты).
- Чем ответ Victoria отличается: недостатки и что улучшить.

Эталоны складывать в **docs/curator_reports/standards/** (см. README и пример what_can_you_do.md). Сравнение ответа с эталоном: `python3 scripts/curator_compare_to_standard.py --report путь/к/отчёту.json --standard имя_эталона`. Использовать для:
- обновления промптов и порогов маршрутизации;
- добавления примеров в knowledge_nodes (обучение корпорации).

**Куратор как регрессия:** регулярный прогон (например по расписанию) + сравнение с эталоном выявляет ухудшения. **Candidate for standard:** если пользователь явно принял ответ (обратная связь «принять»), можно предложить его как кандидат в эталоны — добавить в standards/ и при необходимости в RAG.

## 4. Наставничество: что передать в корпорацию

- [ ] **knowledge_nodes:** добавить сниппеты «Вопрос → эталонный ответ» по итогам прогона.
- [ ] **Промпты:** скорректировать system_prompt или инструкции по категориям (informational, coding, analysis).
- [ ] **Маршрутизация:** уточнить ключевые слова/пороги в task_detector и PREFER_EXPERTS_FIRST.
- [ ] **Документация:** обновить MASTER_REFERENCE, CHANGES, VERIFICATION_CHECKLIST по выявленным причинам сбоев.

## 5. Регулярность

- По запросу: «прогони кураторский набор», «проверь Victoria», «сделай как куратор».
- Опционально: раз в неделю/спринт — прогон списка + FINDINGS + при необходимости обновление эталонов и knowledge_nodes.

---

**Полный прогон (при запущенной Victoria):**
```bash
./scripts/run_curator.sh --file scripts/curator_tasks.txt --async --max-wait 600
```
Или быстрый: `./scripts/run_curator.sh` (2 задачи).
