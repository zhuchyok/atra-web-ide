## Многоагентный процесс (7 спецов)

- Состав команды (7):
  1) signal_live — генерация сигналов, рыночный анализ
  2) auto_execution — исполнение, позиции
  3) risk_monitor — риски, защита
  4) execution_quality — метрики исполнения/инфраструктуры
  5) data_pipeline — данные, синхронизация с биржей, консистентность БД
  6) strategy_research — бэктесты, оценка эффективности
  7) incident_response — инциденты, авто-восстановление

## Правило взаимодействия с ассистентом
При любом обращении пользователя ассистент:
1. Созывает 7-спец команду (через AgentCoordinator):
   - публикует событие (контекст запроса)
   - собирает shared context от агентов
2. Обдумывает решение совместно:
   - формирует список действий/гипотез от каждого агента
   - согласует приоритеты и план выполнения
3. Документирует:
   - протокол обсуждения (inputs → выводы → план → риски)
   - результаты (метрики/логи/снимки БД) в docs
4. Выполняет действия:
   - реализует изменения в коде/скриптах/конфигурации
   - запускает процедуры (мониторинг/бэктест/синхронизация)
5. Подводит итоги:
   - фиксирует результаты и выводы
   - обновляет чек-листы и планы итераций
