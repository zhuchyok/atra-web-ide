# Отчет: Исправление неиспользуемой переменной qty_override

## Проблема
- **Файл:** `telegram_bot.py`
- **Строка:** 1415
- **Ошибка:** Possibly unused variable 'qty_override'
- **Тип:** Warning

## Анализ
Переменная `qty_override` объявлялась и присваивалась значения в callback_data для DCA сигналов, но не использовалась в коде для переопределения расчетного объема позиции.

## Решение

### 1. Добавлено использование переменной
```python
# Объем с учетом плеча
qty = (risk_amount * (dynamic_leverage if trade_mode == "futures" else 1)) / current_price if current_price > 0 else 0

# Если есть переопределение объема (для DCA сигналов), используем его
if qty_override is not None:
    qty = qty_override
```

### 2. Улучшено отображение в сообщении
Добавлена индикация DCA объема в сообщении о принятии сигнала:
```python
msg += f"• **Объём:** `{qty:.4f}`"

# Добавляем информацию о переопределенном объеме для DCA
if qty_override is not None:
    msg += " *(DCA объём)*"
```

## Результат
- ✅ Устранено предупреждение линтера
- ✅ Переменная `qty_override` теперь корректно используется для DCA сигналов
- ✅ Добавлена визуальная индикация DCA объема в сообщениях
- ✅ Синтаксис проверен и корректен

## Функциональность
Теперь при принятии DCA сигналов с переопределенным объемом:
1. Объем позиции будет установлен согласно значению из callback_data
2. В сообщении будет отмечено, что используется DCA объем
3. Система корректно обрабатывает как обычные, так и DCA сигналы
